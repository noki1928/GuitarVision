# GuitarVision — Анализ хвата медиатора в реальном времени

## 🚀 Быстрый старт

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Запуск веб-сервиса (рекомендуется)

```bash
python app.py                          # Без эталона
python app.py reference_grip.jpg       # С эталонным хватом из фото
```

Откройте в браузере: **http://localhost:5000**

### Запуск десктопного приложения

```bash
python hand_tracking.py                # Классификация с анализом хвата
python grip_classifier.py              # Только классификатор
```

**Управление в десктопном приложении:**
- `q` — выход
- `s` — сделать скриншот
- `a` — включить/выключить звуковую обратную связь

## 📋 Что делает система

`GuitarVision` анализирует хват медиатора в реальном времени и предоставляет мгновенную обратную связь:

### 1. **Отслеживание кисти (HandTracker)**
- Обнаружение и трекинг 21 landmark на руке через Mediapipe
- Визуализация скелета руки поверх видео
- Определение положения кончиков пальцев

### 2. **Классификация хвата (PickGripClassifier)**
Система определяет тип хвата и даёт рекомендации:

#### Типы хвата:
- ✅ **Correct Grip** — хват совпадает с эталоном
- ❌ **Too Tight** — медиатор зажат слишком сильно
- ⚠️ **Too Loose** — медиатор может выскользнуть
- 📐 **Wrong Pinch Angle** — угол между пальцами не оптимален
- 🖐️ **Fingers Not Curved** — пальцы раскрыты (неправильное положение)
- 👍 **Wrong Thumb Position** — неправильное положение большого пальца
- 😰 **Hand Tension** — напряжение в кисти

### 3. **Веб-интерфейс**
- Видеопоток с камеры в реальном времени
- Панель анализа с метриками
- Настройка строгости проверки (tolerance)
- Рекомендации по исправлению хвата
- Визуализация кривизны каждого пальца

### 4. **Звуковая обратная связь** (десктоп)
- Звуковые сигналы при изменении типа хвата
- Разные тона для разных проблем
- Позволяет сосредоточиться на игре, не глядя на экран

## 🎸 Метрики анализа

### Ключевые параметры

#### 1. **Расстояние щипка (Pinch Distance)**
Расстояние между кончиками большого и указательного пальцев.

Сравнивается с эталонным изображением (если загружено).

#### 2. **Угол щипка (Pinch Angle)**
Угол между большим и указательным пальцами.

#### 3. **Угол большого пальца (Thumb Angle)**
Определяет, насколько согнут большой палец.

#### 4. **Кривизна пальцев (Finger Curvature)**
Степень сгиба каждого пальца:
- Средний, безымянный и мизинец должны быть согнуты
- Визуальная индикация: красный (прямой) → оранжевый (средний) → зелёный (согнут)

---

## 🎯 Настройка строгости проверки (Tolerance)

В веб-интерфейсе доступен слайдер **Strictness Level**, который позволяет настроить насколько строго проверять хват:

| Значение | Режим | Описание |
|----------|-------|----------|
| 5-10% | Очень строго | Хват должен почти идеально совпадать с эталоном |
| 15-20% | Строго | Допускаются небольшие отклонения |
| 25-35% | Средне | Значительные отклонения всё ещё приемлемы |
| 40-50% | Мягко | Свободная проверка |

По умолчанию: **15%** (строгая проверка)

## 🔧 Как это работает

### Загрузка эталонного хвата

При запуске с фотографией эталона:
```bash
python app.py reference_grip.jpg
```

Система:
1. Загружает фотографию правильного хвата
2. Обнаруживает руку на фото
3. Извлекает метрики (расстояние между большим и указательным пальцами)
4. Использует эти метрики как эталон для сравнения

### Классификация

Если эталон загружен:
- Сравнивается **только расстояние pinch** с эталоном
- Угол щипка не учитывается
- Отклонение в пределах tolerance → "Correct Grip"
- Отклонение超出 tolerance → "Too Tight" или "Too Loose"

Если эталон не загружен:
- Используются стандартные эмпирические пороги
- Проверка мягче, только базовые метрики
