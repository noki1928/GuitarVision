## Описание проекта

**GuitarVision** - система анализа хвата медиатора в реальном времени с использованием компьютерного зрения.

### Основные возможности

- Real-time анализ положения кисти через веб-камеру
- Масштабо-инвариантная классификация (не зависит от расстояния до камеры)
- 4 типа классификации хвата + определение отсутствия руки
- Современный веб-интерфейс с природной цветовой палитрой
- Двухколоночная компоновка (видео слева, анализ справа)
- Десктопное приложение со звуковой обратной связью
- Настраиваемые пороги классификации
- Видео формат 16:9 с автоматической обрезкой

### Технологии

- **Python 3.12**
- **MediaPipe** - детекция landmarks кисти (21 точка)
- **OpenCV** - обработка видео
- **Flask** - веб-сервис
- **NumPy** - математические вычисления

### Компоненты системы

#### 1. HandTracker (`hand_tracking.py`)

**Назначение:** Детекция и отслеживание кисти руки через MediaPipe.

**Ключевые методы:**
- `find_hands(frame, draw=True)` - детекция landmarks на кадре
- `get_cached_landmarks()` - получение кэшированных landmarks (без повторной детекции)
- `get_finger_tips(landmarks)` - координаты кончиков пальцев

**Оптимизация:**
- Кэширование результатов MediaPipe для избежания дублирования детекции
- Режим `RunningMode.IMAGE` для синхронной обработки

#### 2. PickGripClassifier (`grip_classifier.py`)

**Назначение:** Классификация хвата на основе геометрии кисти.

**Ключевые методы:**
- `load_reference_image(path)` - загрузка эталонного изображения
- `analyze_grip(landmarks)` - анализ текущего хвата
- `_classify()` - логика классификации

**Метрики:**
- `hand_size` - размер ладони (запястье → средний палец MCP)
- `normalized_pinch` - нормализованное расстояние щипка
- `avg_curl` - средний сгиб пальцев (средний, безымянный, мизинец)
- `rotation_vector` - вектор поворота кисти (запястье → указательный MCP)

#### 3. Flask Web Service (`app.py`)

**Назначение:** Веб-интерфейс с видеопотоком и API.

**Endpoints:**
- `GET /` - главная страница
- `GET /video_feed` - MJPEG видеопоток
- `GET /api/analysis` - JSON с результатами анализа
- `POST /api/tolerance` - установка порога tolerance

**Архитектура:**
- Многопоточность: видеопоток в отдельном потоке
- Синхронизация через `frame_lock`
- Polling интервал: 100ms

## Алгоритм классификации

### 5 состояний системы

1. **CORRECT (Правильно)** - все метрики в пределах допуска
2. **ROTATED (Кисть повернута)** - угол отклонения >40°
3. **TOO_TIGHT (Пальцы сильно сжаты)** - avg_curl < reference × 0.5
4. **FINGERS_OPEN (Пальцы разжаты)** - avg_curl > reference × 2.5
5. **NO_HAND (Кисть не обнаружена)** - рука не видна в кадре

### Приоритет проверок

```
┌─────────────────────────────────────────┐
│ ПРИОРИТЕТ 1: Угол поворота кисти        │
│ Порог: 40°                              │
│ Если отклонение > 40° → ROTATED         │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│ ПРИОРИТЕТ 2: Сгиб пальцев               │
│ (средний, безымянный, мизинец)          │
│                                         │
│ avg_curl < ref × 0.5 → TOO_TIGHT        │
│ ref × 0.5 ≤ avg_curl ≤ ref × 2.5        │
│                      → CORRECT          │
│ avg_curl > ref × 2.5 → FINGERS_OPEN     │
└─────────────────────────────────────────┘
```

### Асимметричное окно правильности

**Для эталона `avg_curl = 0.580`:**

| Зона | Диапазон | Tolerance | Результат |
|------|----------|-----------|-----------|
| Слишком сжато | < 0.290 | 50% | TOO_TIGHT |
| **Окно правильности** | **0.290 - 1.450** | - | CORRECT |
| Слишком разжато | > 1.450 | 150% | FINGERS_OPEN |

**Ширина окна:** 200% от эталона

## Метрики и нормализация

### Масштабо-инвариантность

Все расстояния нормализуются по размеру ладони:

```python
hand_size = distance(wrist[0], middle_mcp[9])
normalized_metric = raw_metric / hand_size
```

**Преимущества:**
- Не зависит от расстояния до камеры
- Не зависит от размера руки
- Стабильные пороги для всех пользователей

### Ключевые точки MediaPipe

```
Индексы landmarks (всего 21 точка):
  0  = wrist (запястье)
  4  = thumb_tip (большой палец)
  5  = index_mcp (костяшка указательного)
  8  = index_tip (указательный палец)
  9  = middle_mcp (костяшка среднего)
  12 = middle_tip (средний палец)
  13 = ring_mcp (костяшка безымянного)
  16 = ring_tip (безымянный палец)
  17 = pinky_mcp (костяшка мизинца)
  20 = pinky_tip (мизинец)
```

### Расчет метрик

#### 1. Размер ладони (hand_size)

```python
hand_size = distance(wrist[0], middle_mcp[9])
```

#### 2. Сгиб пальцев (avg_curl)

```python
middle_curl = distance(middle_tip[12], middle_mcp[9]) / hand_size
ring_curl = distance(ring_tip[16], ring_mcp[13]) / hand_size
pinky_curl = distance(pinky_tip[20], pinky_mcp[17]) / hand_size

avg_curl = (middle_curl + ring_curl + pinky_curl) / 3
```

**Важно:** Большой и указательный пальцы НЕ учитываются (держат медиатор).

#### 3. Поворот кисти (rotation_vector)

```python
rotation_vector = index_mcp[5] - wrist[0]
rotation_angle = arctan2(vector.y, vector.x)

# Сравнение через скалярное произведение
cos_angle = dot(ref_vector, current_vector) / (norm(ref) * norm(current))
rotation_deviation = arccos(cos_angle)
```

## API и интеграция

### REST API

#### GET /api/analysis

Возвращает текущий результат анализа.

**Response:**
```json
{
  "status": "correct",           // CSS класс: correct/warning/error
  "grip_type": "Правильно",      // Тип хвата
  "confidence": 90.0,            // Уверенность (0-100%)
  "pinch_distance": 40.3,        // Расстояние щипка (пиксели)
  "hand_rotation": 174.6,        // Угол поворота (градусы)
  "recommendation": "Отлично!",  // Рекомендация
  "timestamp": 1234567890.123,   // Unix timestamp
  "age_ms": 50                   // Задержка анализа (мс)
}
```

**Response (когда рука не обнаружена):**
```json
{
  "status": "error",
  "grip_type": "Кисть не обнаружена",
  "confidence": 0,
  "pinch_distance": 0,
  "hand_rotation": 0,
  "recommendation": "Поместите руку в кадр камеры",
  "timestamp": 0,
  "age_ms": 0
}
```

#### POST /api/tolerance

Установка порога tolerance.

**Request:**
```json
{
  "tolerance": 0.25  // 0.05 - 0.50
}
```

**Response:**
```json
{
  "status": "success",
  "tolerance": 0.25,
  "message": "Tolerance set to 25%"
}
```


## Оптимизация производительности

### Достигнутые результаты

| Метрика | До оптимизации | После оптимизации | Улучшение |
|---------|----------------|-------------------|-----------|
| Задержка на кадр | ~396ms | ~80-100ms | 4-5x |
| Эффективный FPS | ~2.5 | ~10-12 | 4-5x |
| Polling интервал | 500ms | 100ms | 5x |

### Ключевые оптимизации

#### 1. Кэширование MediaPipe результатов

**Проблема:** Двойная детекция (find_hands + get_landmarks)

**Решение:**
```python
# HandTracker.find_hands() кэширует результаты
self.cached_landmarks = landmarks

# Повторное использование без детекции
landmarks = tracker.get_cached_landmarks()
```

**Выигрыш:** ~40ms на кадр

#### 2. Одно чтение камеры

**Проблема:** Двойное чтение кадра

**Решение:**
```python
# Было:
ret, frame = camera.read()  # Чтение #1
ret, frame = camera.read()  # Чтение #2

# Стало:
ret, frame = camera.read()  # Одно чтение
```

**Выигрыш:** ~33ms на кадр

#### 3. Удаление избыточных вычислений

**Удалено из analyze_grip():**
- finger_curvature для всех 5 пальцев (~10 вычислений углов)
- thumb_position_score
- hand_tension
- pinch_angle, thumb_angle (не используются)

**Выигрыш:** ~10-15ms на кадр

#### 4. Уменьшение polling интервала

**Изменение:** 500ms → 100ms

**Выигрыш:** ~200ms средней задержки отображения

## Решение проблем

### Проблема: "No hand detected"

**Причины:**
- Рука не в кадре
- Плохое освещение
- Рука слишком далеко от камеры

**Решение:**
- Поднесите руку ближе к камере
- Улучшите освещение
- Убедитесь, что вся кисть видна

### Проблема: Постоянно показывает "Кисть повернута"

**Причина:** Положение кисти сильно отличается от эталона

**Решение:**
- Создайте новое эталонное изображение в вашем текущем положении
- Или увеличьте `rotation_threshold` в `grip_classifier.py`

### Проблема: Низкий FPS

**Причины:**
- Слабый процессор
- Другие приложения используют камеру
- Высокое разрешение камеры

**Решение:**
```python
# Уменьшите разрешение камеры в app.py
camera.set(cv2.CAP_PROP_FRAME_WIDTH, 640)
camera.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)
```

### Проблема: Зеркальное отображение неправильное

**Решение:**
```python
# Отключите flip в app.py (строка 69)
# frame = cv2.flip(frame, 1)  # Закомментируйте эту строку
```

### Проблема: "hands.task not found"

**Причина:** Модель MediaPipe отсутствует

**Решение:**
- Скачайте модель с [MediaPipe Models](https://developers.google.com/mediapipe/solutions/vision/hand_landmarker#models)
- Поместите файл `hand_landmarker.task` в корень проекта
- Переименуйте в `hands.task`

### Проблема: Звук не работает (десктопное приложение)

**Причина:** `winsound` работает только на Windows

**Решение:**
- На Linux/macOS звук заменяется на print в консоль
- Или отключите звук клавишей `a`
